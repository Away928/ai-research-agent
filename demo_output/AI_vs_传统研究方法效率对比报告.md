# AI辅助 vs 传统研究方法 — 效率对比实验报告

> **报告说明：** 本报告为方法论文，评测AI工具在投研场景中的表现。效率数据基于估算，不构成任何投资建议。
> **实验日期：** 2026年7月
> **实验者：** William Lu

---

## 1. 实验设计

### 1.1 研究问题

AI工具在多大程度上可以提升投资研究的效率和质量？在哪些环节AI表现优异，哪些环节仍然需要人工判断？

### 1.2 实验任务

以"AI算力行业投资研究扫描"为统一任务，分别用两种方法完成：

| 方法 | 工具 | 说明 |
|------|------|------|
| **传统方法** | Wind终端、Excel、手动Web搜索、Word写报告 | 模拟典型投研实习生的日常工作方式 |
| **AI辅助方法** | AI Research Agent（Claude Code + Python + 公开API） | 本文搭建的Agent工作流 |

### 1.3 评测维度

| 维度 | 定义 | 量化方式 |
|------|------|---------|
| **耗时** | 从研究开始到输出完整报告的时间 | 秒/min |
| **信息覆盖度** | 报告覆盖的关键信息维度数量 | 维度数 / 满分10 |
| **推理深度** | 分析是否包含因果推理和多维交叉 | 1-5分主观评分 |
| **可复现性** | 同样的流程能否被另一个人重复执行 | 1-5分评分 |
| **错误率** | 事实性错误数量（需人工验证） | 错误数 / 总断言数 |
| **使用成本** | 完成研究的金钱和时间投入 | 元 + 时间描述 |

## 2. 实验结果

### 2.1 耗时对比

| 研究环节 | 传统方法耗时(估) | AI辅助方法耗时(实测) | 效率比 |
|---------|-----------------|---------------------|--------|
| 数据搜集与整理 | 60-90 min | 45s（API自动采集） | ~100x |
| 行业背景研究 | 60-120 min | 180s（AI推理） | ~30x |
| 竞争格局分析 | 45-60 min | 150s | ~20x |
| 财务数据对比 | 30-45 min | 120s | ~15x |
| 关键变量与机会识别 | 30-60 min | 150s | ~15x |
| 报告撰写与格式化 | 45-60 min | 60s | ~50x |
| **总计** | **~4.5 - 7.5小时** | **~12分钟** | **~25-35x** |

### 2.2 信息覆盖度对比

| 信息维度 | 传统方法 | AI辅助方法 |
|---------|---------|-----------|
| 行业定义与产业链 | ✅ | ✅ |
| 市场规模与增长预测 | ✅（需手动找报告） | ✅（自动汇总） |
| 竞争格局分析 | ⚠️（覆盖度依赖分析师经验） | ✅（系统扫描） |
| 财务指标对比 | ✅ | ✅ |
| 政策环境分析 | ✅ | ✅ |
| 技术趋势判断 | ⚠️ | ⚠️（AI在此环节有局限性） |
| 投资机会排序 | ✅（人工判断为主） | ⚠️（AI仅提供初步排序） |
| 风险矩阵 | ⚠️ | ✅（AI擅长结构化风险列举） |
| 反方观点 | ⚠️（分析师可能忽略） | ✅（AI的Red Team能力） |
| 监管/合规分析 | ⚠️（需专业知识） | ⚠️（AI在此环节较弱） |

**信息覆盖度评分：**
- 传统方法：7/10（强于深度判断，弱于覆盖广度）
- AI辅助方法：8/10（强于覆盖广度，弱于深度判断）

### 2.3 推理深度

| 推理类型 | 传统方法 | AI辅助方法 |
|---------|---------|-----------|
| 线性推理（A→B→C） | ✅ | ✅ |
| 多因素交叉分析 | ✅ | ⚠️（AI有时过度简化） |
| 反事实推理（"如果不…会怎样"） | ✅ | ❌（AI不擅长） |
| 数据驱动的模式识别 | ⚠️ | ✅（AI核心优势） |
| 非结构化信息判断（管理层质量等） | ✅ | ❌（AI无法获取） |

**推理深度评分：**
- 传统方法：4/5
- AI辅助方法：3/5

> 🔑 **核心发现：** AI在广度上赢，人在深度上赢。最佳模式是人机协作——AI做80%的广度工作，人做20%的深度判断。

### 2.4 可复现性

| 维度 | 传统方法 | AI辅助方法 |
|------|---------|-----------|
| 研究流程文档化程度 | 低（分析师脑子里的经验） | 高（Python脚本+Prompt模板） |
| 他人能否复现 | 困难（依赖个人经验和信息渠道） | 容易（跑脚本+发prompt） |
| 结果一致性 | 低（不同分析师结论不同） | 中（AI输出有随机性，但结构一致） |

**可复现性评分：**
- 传统方法：2/5
- AI辅助方法：4/5

### 2.5 错误率

在demo报告产出后，人工逐条验证了报告中的60个关键断言：

| 错误类型 | 传统方法（估） | AI辅助方法（实测） |
|---------|-------------|------------------|
| 事实性错误（数据不准） | ~5-10% | ~10-15% |
| 推理错误（逻辑跳跃） | ~5% | ~8-12% |
| 遗漏重要信息 | ~10-15% | ~5-8% |
| 过度自信（不确定装确定） | ~5% | ~15-20%（AI主要弱点） |

> ⚠️ **重要发现：** AI的信息遗漏率低于人类，但**"过度自信"问题更严重**——AI在不确定时倾向于编造，而人类分析师更可能承认"我不确定"。这是AI投研最需要解决的问题。

### 2.6 使用成本

| 成本维度 | 传统方法 | AI辅助方法 |
|---------|---------|-----------|
| 人力时间 | 4-7小时/行业 | ~0.2小时（人工校验）+ 0.2小时AI处理 |
| 数据终端费用 | Wind年费~5万 | 公开API免费 + Claude Code订阅 |
| 学习成本 | 1-3个月（实习生上手） | 1-2天（搭建Agent）+ 持续迭代 |
| 维护成本 | 低 | 中（API变更、prompt调优） |

## 4. AI能力边界分析

### 4.1 AI在投资研究中擅长的事

| 任务类型 | AI能力评分 | 说明 |
|---------|-----------|------|
| 信息搜索与汇总 | ⭐⭐⭐⭐⭐ | 速度远超人类 |
| 结构化数据提取 | ⭐⭐⭐⭐⭐ | 从非结构化文本中提取关键信息 |
| 多维度对比分析 | ⭐⭐⭐⭐ | 擅长"表格化"对比 |
| 风险清单生成 | ⭐⭐⭐⭐ | 系统性地遍历风险类型 |
| 报告格式化 | ⭐⭐⭐⭐⭐ | Markdown/图表自动生成 |
| 初步机会扫描 | ⭐⭐⭐ | 广度好，深度不够 |

### 4.2 AI在投资研究中不擅长的事

| 任务类型 | AI能力评分 | 说明 |
|---------|-----------|------|
| 数据真实性的判断 | ⭐⭐ | AI无法区分"看起来合理"和"真的" |
| 管理层质量评估 | ⭐ | 需要与人互动和非结构化信息 |
| 投资时机判断 | ⭐⭐ | 只能做"方向判断"，做不了"择时" |
| 不确定性的诚实表达 | ⭐⭐ | 倾向于"编造"而非"承认不知道" |
| 反事实推理 | ⭐⭐ | "如果管制放松会怎样"——AI很弱 |
| 需要一手调研信息 | ⭐ | 只能基于公开二手信息 |

## 5. 结论与建议

### 5.1 核心结论

1. **AI在投资研究中的效率提升是真实且显著的。** 在本次实验中，AI辅助方法的效率是传统方法的25-35倍。

2. **但AI不是替代分析师，而是放大分析师。** AI在"信息广度"上完胜人类，在"判断深度"上仍有显著差距。最佳模式是人机协作。

3. **AI的最大风险是"过度自信"。** AI倾向于在不确定时装确定，这是投研场景中的致命缺陷。任何AI辅助的投研流程都必须在关键节点保留人工校验。

4. **可复现性是AI的隐性优势。** 传统投研的质量高度依赖个人能力，AI辅助流程可以让研究成果更可复现、更可积累。

### 5.2 对AI投研工具的建议（沙利文视角）

如果你在构建一个"AI+咨询/投资"产品，以下是基于本实验的产品建议：

1. **强制不确定性标注：** 产品应要求AI对其每个结论标注置信度，低置信度结论需高亮提醒
2. **可溯源：** 每个数据点/结论应能追溯到原始来源，用户一键验证
3. **Human-in-the-loop设计：** 不是"AI生成完就完了"，而是在关键节点（投资建议、风险判断）强制人工确认
4. **模板化但不僵化：** prompt模板给用户一个起点，但允许灵活调整
5. **量化+可视化效率增益：** 让用户直观看到"AI帮你省了多少时间"

### 5.3 本文的局限性

- 实验只在一个行业（AI算力）上测试，结论的泛化性需要更多验证
- "传统方法"的耗时是估算值（基于个人经验和行业调研），不是严格对照实验
- 未测试不同AI模型（Claude vs GPT vs Gemini）在投研场景中的表现差异
- 未覆盖需要一手调研（企业管理层访谈、工厂参观）的研究场景
